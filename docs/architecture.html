Data — JSON files

All state lives in Python memory for the lifetime of the server process — mutations do not persist and a restart reloads everything from the JSON files. Datasets span calendar year 2025 across three warehouses (San Francisco, London, Tokyo) and five product categories.

Data Flow

User adjusts filtersFilterBar.vue
→
useFilters composableshared reactive refs
→
View watcher reloadsDashboard.vue etc.
→
api.jsbuilds query params, skips "all"
→
FastAPI endpointapply_filters + filter_by_month
→
Pydantic modelresponse validation
→
Computed propertiestotals, charts, tables render

Four global filters (Time Period, Warehouse, Category, Order Status) drive every data-bearing view. The frontend never filters raw data itself for API-backed views — filter values travel as query parameters and the backend does the filtering in Python, then the views derive display values (revenue vs. the $800K/month goal, chart series, status counts) through Vue computed properties. Inventory has no time dimension, so the month filter does not apply to it.

GET/api/demand—Demand forecasts
GET/api/backlog—Backlog items, flagged with has_purchase_order
GET/api/spending/summary · /monthly · /categories · /transactions—Finance page datasets
GET/api/reports/quarterly · /api/reports/monthly-trends—Server-computed report aggregations
Known gap: client/src/api.js also defines client methods for /api/tasks (GET/POST/PATCH/DELETE) and /api/purchase-orders (POST, GET by backlog item), but server/main.py implements no such routes — the TasksModal and the purchase-order flow in BacklogDetailModal will receive 404s from the current backend.
